    • Trees, GraphTrees are the most common structure to represent hierarchies as they allow placement of data in a parent-child relationship. Different varieties of trees have their unique set of operations and properties which make them suitable for different scenarios. Below is an outline of how and why these structures cater and optimize the challenges: 1. *General Tree*: - It allows storing data in hierarchy but without having a rigid structure - Suitable for more complex hierarchies such as organizational charts, file systems and so on - Does not enhance capabilities for some operations such as search or balance 2. *Binary Search Tree (BST)*: - The left node is a child of the parent that is lesser than the parent, and the right node it greater than the parent. - Useful for systems where the datawords are volatile and require a lot of search, insertions and deletions. - However, they can become lopsided which can reduce their efficiency in the worst instances potentially to O(n). 3. *AVL Tree*: - It is a specific type of binary search tree where the height of the left and right sub trees cannot differ by more than 1. - A search, insert, and delete operations in an AVL tree would roughly have a time complexity of O(log n). - Especially useful in situations where constant speed of searching for data is necessary for example a database. 4. *2-3 Trees*: - It is a multi way search tree where a node can have either 1 or 2 keys but remains self balancing. - 2-3 trees are low height trees thus allowing O(log n) runtime for any kind of operation.
    • Algorithms: Kruskal's Algorithm:Greedy approach which chooses the smallest edges in non-decreasing order, so no cycles are formed. Best for sparse graphs (few edges). Prim's Algorithm: Greedy approach that grows the spanning tree by choosing the smallest edge connecting a vertex in the tree to one not in the tree. Best for dense graphs (many edges). Dijkstra's Algorithm: Greedy approach that computes the shortest path from a source to all other nodes. Works best for graphs with non-negative weights. Bellman-Ford Algorithm: Iterative approach that computes shortest paths, even with negative edge weights. The value really comes when detecting negative weight cycles. Floyd-Warshall Algorithm: DP solution to find shortest paths between all pairs of nodes. Effective when dealing with small to medium-sized dense graphs.
